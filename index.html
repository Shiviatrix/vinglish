02 / Types & methods

Data has shape. Behavior has a home.

Use type for records, name: type for fields, and braces for construction. Use on to give a type its own behaviour.

type Point
begin
    x: number
    y: number
end

function translate on Point (number dx, number dy) returns Point
begin
    return Point { x: self.x + dx, y: self.y + dy }
end

03 / Generics

Reusable and explicit.

A Vector of T is written Vector<T> in source. Generic calls remain explicit and statically checked, without a dynamic dispatch layer.

use std.collections.vector

let values be vector_new<number>()
push(borrow mutable values, 42)
println(get(borrow values, 0))
vector_free(borrow mutable values)

04 / Error handling

Failure is explicit in the type.

Result of T makes recoverable failure visible. Return Ok(value) or Err(message); use ? to pass the error through an operation that can also fail.

function connect() returns Result of number
begin
    let socket be tcp_connect("127.0.0.1", 8080)?
    tcp_close(borrow socket)
    return Ok(0)
end

Explicit by design: an error cannot disappear by accident when the caller has to produce another Result of T.

05 / Compiler pipeline

Validated source. Optimized native output.

The compiler resolves a program into HIR, lowers it to SSA-form MIR, then applies constant propagation, GVN, DCE, and control-flow simplification before code generation.

HIR

Names, types, imports, ownership, and semantic validity.

MIR + SSA

Clear control flow and single-assignment data flow.

GVN + DCE

Remove redundant calculations and code without observable effect.

C backend

Transparent C output plus a minimal runtime.
