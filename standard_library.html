01 / MODULE MAP

A focused native layer.

std/
├── collections/vector.ving
├── collections/map.ving
├── io.ving · string.ving · math.ving
├── file.ving · net.ving · thread.ving
└── ui.ving · runtime.ving

Import modules by dotted path. Public declarations cross the module boundary; implementation details remain local.

02 / COLLECTIONS

Vector<T> and Map

use std.collections.vector
let scores be vector_new<number>()
push(borrow mutable scores, 42)
vector_free(borrow mutable scores)

Collections own their allocations. Release allocation-backed values exactly once when they are no longer needed.

03 / I/O & TEXT

Console input, output, and formatting.

use std.io
let user be read_line()
println(fmt!("Hello, {user}."))

std.io supplies console and basic text helpers; std.string contains allocation-backed string operations.

04 / NETWORKING

Failures remain explicit.

std.net wraps TCP sockets in TcpSocket. Connect, send, and receive operations return Result of T.

05 / NATIVE UI

Desktop windows without hidden glue.

std.ui exposes windows, buffers, pixels, and input through Vinglish foreign declarations backed by the Rust runtime bridge.
